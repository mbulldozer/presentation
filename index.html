Static Types

TypeScript adds optional static types to JavaScript, this means that you can declare a typed variable:

let variableName: typeScriptType = value;

And the type of this variable will be the same in your development process thanks to the type checking at compile time:

let a: string = “Hello TypeScript!”; // string
a = 0; // Error!

The transpiler would alert us of this error with a message like: Type “number” is not assignable to type” string”.

Functions

TypeScript adds new capabilities to these functions, one of them being the capability to declare the type of each parameter in order to create the signature:

function add(x: number, y: number) {
  return x + y;
}

We can also add a return type, but TypeScript is able to infer it with the return statements:

function add(x: number, y: number): number {
  return x + y;
}

Optional and Default Parameters

Optional:

function buildName(firstName: string, lastName?: string) {
  if (lastName)
	return firstName + " " + lastName;
  else
	return firstName;
}

Default:

function buildName(firstName: string, lastName = "Smith") {
	return firstName + " " + lastName;
}

Function Overloads

TypeScript lets us specify an overload list to supply multiple types for the same function. That way, we can describe to the type system exactly what our function accepts and what it returns:

function addOvverload(x: string, y: string): string;
function addOvverload(x: number, y: number): number;
function addOvverload(x, y): any {
  return x + y;
}

Classes

In general, a class is composed of a name, attributes and methods. The syntax to create a simple class looks like this:

class Person {
  name: string;
							
  constructor(name : string) {
	this.name = name;
  }
							
  greet() {
	console.log("Hi! I'm " + this.name);
  }
}

Inheritance

class Animal {
	move(distanceInMeters: number = 0) {
		console.log(`Animal moved ${distanceInMeters}m.`);
	}
}
						
 class Dog extends Animal {
	bark() {
		console.log('Woof! Woof!');
	}
}
						
const dog = new Dog();
